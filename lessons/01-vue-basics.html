Vue.js พื้นฐาน

โปรเจกต์ KL.Core.UI ใช้ Vue 2 (เวอร์ชัน 2.7.14) แบบ Options API — ไม่ใช่ Vue 3 และไม่ใช่ Composition API ดังนั้นบทเรียนนี้จะสอน Vue 2 Options API ล้วน ๆ ให้แน่นก่อน ตัวอย่างในหน้านี้เป็น Vue จริงที่รันอยู่ในเบราว์เซอร์ (โหลดจาก CDN) ลองแก้ไข/กดปุ่มดูได้เลย

1. Vue instance คืออะไร

Vue app คือ object ที่บอกว่า "ข้อมูล (data) มีอะไรบ้าง" และ "เอาข้อมูลไปแสดงยังไง (template)" เมื่อข้อมูลเปลี่ยน หน้าจอจะอัปเดตให้อัตโนมัติ (reactivity) โดยที่เราไม่ต้องไปสั่ง document.getElementById(...).innerText = ... เอง 

new Vue({
  el: "#app",           // ผูกกับ element ไหนใน HTML
  data() {               // ข้อมูลของ component
    return {
      message: "สวัสดี Vue"
    };
  }
});

3. data, computed, methods, watch

  • data — ตัวแปร/state ของ component (ต้องเป็น function ที่ return object เสมอใน component — ต่างจาก root instance ที่จะใส่ object ตรง ๆ ก็ได้)
  • computed — ค่าที่ "คำนวณจาก data อื่น" และจะถูก cache ไว้ จะคำนวณใหม่ก็ต่อเมื่อ dependency เปลี่ยนเท่านั้น (เร็วกว่า method ถ้าถูกเรียกใช้ซ้ำ ๆ ใน template)
  • methods — ฟังก์ชันทั่วไป เรียกเมื่อไหร่ก็ทำงานใหม่ทุกครั้ง เหมาะกับ event handler
  • watch — ดักฟังว่า data/computed ตัวไหนเปลี่ยนค่า แล้วสั่งทำ side-effect (เช่น เรียก API, เขียน log)
export default {
  data() {
    return { price: 100, qty: 2 };
  },
  computed: {
    total() {                      // ใช้ this.total ใน template ได้เลย เหมือนตัวแปร
      return this.price * this.qty;
    }
  },
  methods: {
    increaseQty() {                // ใช้ this.increaseQty() หรือ @click="increaseQty"
      this.qty++;
    }
  },
  watch: {
    qty(newVal, oldVal) {
      console.log(`qty เปลี่ยนจาก ${oldVal} เป็น ${newVal}`);
    }
  }
};

4. Component — แตกหน้าจอเป็นชิ้นเล็ก ๆ

Component คือ Vue instance ย่อยที่ใช้ซ้ำได้ มีของตัวเอง (data/methods) แต่คุยกับข้างนอกผ่าน 2 ทางเท่านั้น:

ลงมา (Parent → Child)

Props

Parent ส่งข้อมูลลงมาให้ Child ผ่าน attribute — Child ห้ามแก้ props ตรง ๆ (ให้ emit ขึ้นไปให้ parent แก้แทน)

ขึ้นไป (Child → Parent)

Events ($emit)

Child ยิง event บอก parent ว่า "มีอะไรเกิดขึ้น" parent ฟังด้วย @event-name

// Child.vue
<template>
  <button @click="$emit('save', localValue)">บันทึก</button>
</template>
<script>
export default {
  props: {
    initialValue: { type: String, default: "" }
  },
  data() {
    return { localValue: this.initialValue };
  }
};
</script>

// Parent.vue
<template>
  <Child :initial-value="name" @save="onChildSave" />
</template>
<script>
import Child from "./Child.vue";
export default {
  components: { Child },
  data() { return { name: "สมชาย" }; },
  methods: {
    onChildSave(value) {
      console.log("child ส่งค่ามาว่า:", value);
    }
  }
};
</script>
กฎเหล็ก: ห้ามแก้ prop โดยตรงใน child
props: ['value'] แล้วไปเขียน this.value = 'x' ใน child — Vue จะเตือน (mutating a prop) เพราะข้อมูลควรไหลทางเดียวจาก parent เท่านั้น ถ้าต้องแก้ ให้ copy มาเก็บใน data ของตัวเอง แล้ว emit event กลับไปให้ parent เป็นคนอัปเดตค่าจริง (โปรเจกต์นี้ใช้ Vuex/pathify แก้ปัญหานี้แบบเป็นระบบ จะเจอในบทที่ 04 และ 07)

5. Slot — ให้ parent "ยัด" เนื้อหาเข้าไปใน component ลูก

// Card.vue (child)
<template>
  <div class="card">
    <slot></slot>          <!-- ใส่ default content ตรงนี้ -->
    <slot name="footer"></slot>  <!-- named slot -->
  </div>
</template>

// ใช้งาน
<Card>
  <p>เนื้อหาหลัก</p>
  <template #footer>
    <button>ปิด</button>
  </template>
</Card>

โปรเจกต์นี้ใช้ named slot เยอะมากในตัว shared component เช่น GridProxy มี slot name="btnOptionR" และ slot name="footerToolbar" ให้แต่ละหน้าจอใส่ปุ่มพิเศษของตัวเองเข้าไปได้ (ดูบทที่ 08)

สรุปสิ่งที่ต้องจำ

  • Vue 2 Options API: component เป็น object ที่มี data / computed / methods / watch / props
  • {{ }} แสดงค่า, : ผูก attribute, @ ดัก event, v-model ผูก 2 ทาง
  • v-if ถอด/สร้าง DOM จริง ๆ, v-show แค่ซ่อนด้วย CSS
  • v-for ต้องมี :key เสมอ
  • ข้อมูลไหลลงด้วย props ไหลขึ้นด้วย $emit ห้ามสวนทาง
