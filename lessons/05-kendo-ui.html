Kendo UI for Vue

Kendo UI คือชุด UI component สำเร็จรูป (ของบริษัท Progress/Telerik ต้องมี license) ที่ให้ Grid, DropDown, DatePicker, Window (dialog), TreeView ที่ครบเครื่องกว่า HTML ธรรมดามาก — โปรเจกต์นี้ใช้ Kendo UI เป็น UI component หลักเกือบทั้งระบบ (ไม่มี live demo ในหน้านี้เพราะ Kendo ต้องใช้ license/CSS ของจริง แต่โค้ดตัวอย่างทั้งหมดคัดมาจากไฟล์จริงในโปรเจกต์)

มี 2 เจนเนอเรชันปนกันอยู่ในโปรเจกต์นี้ — อย่าสับสน
  1. Kendo UI ตัวเก่า (jQuery-based) + Vue Wrapper — package ชื่อ @progress/kendo-*-vue-wrapper (เวอร์ชัน 2020.1.x) ใช้ tag แบบ <kendo-grid>, <kendo-combobox>, <kendo-datepicker> ฯลฯ — เป็นของหลักที่ใช้ 90% ของโปรเจกต์
  2. Kendo UI for Vue ตัวใหม่ (native Vue component) — package ชื่อ @progress/kendo-vue-* (เวอร์ชัน 1.0.3) ใช้แบบ import component ตรง ๆ เช่น import { Grid } from '@progress/kendo-vue-grid' ถูก register เป็น global component ชื่อ <grid> ใน CreateVueInstance.js

1. การติดตั้ง/เปิดใช้งานระดับแอป (ทำไว้แล้ว ไม่ต้องทำซ้ำ)

ทุก component Kendo ถูก register เป็น global component ให้แล้วที่ src/CreateVueInstance.js — เวลาสร้างหน้าจอใหม่ ใช้ tag ได้เลยโดยไม่ต้อง import

// src/CreateVueInstance.js (โค้ดจริงของโปรเจกต์ — ทำไว้ให้แล้ว)
import '@progress/kendo-ui';
import { GridInstaller } from '@progress/kendo-grid-vue-wrapper'
import { DateinputsInstaller } from '@progress/kendo-dateinputs-vue-wrapper'
import { DropdownsInstaller } from '@progress/kendo-dropdowns-vue-wrapper';
import { InputsInstaller } from '@progress/kendo-inputs-vue-wrapper'
import { ButtonsInstaller } from '@progress/kendo-buttons-vue-wrapper'
import { KendoPopupsInstaller } from '@progress/kendo-popups-vue-wrapper'
import { TreeViewInstaller } from '@progress/kendo-treeview-vue-wrapper'
import { UploadInstaller } from '@progress/kendo-upload-vue-wrapper'
// ...ฯลฯ

Vue.use(DateinputsInstaller)
Vue.use(InputsInstaller)
Vue.use(ButtonsInstaller)
Vue.use(GridInstaller)
Vue.use(KendoPopupsInstaller)
Vue.use(DropdownsInstaller)
Vue.use(TreeViewInstaller)
Vue.use(UploadInstaller)
// ทำให้ <kendo-datepicker>, <kendo-combobox>, <kendo-grid> ฯลฯ ใช้ได้ทุกไฟล์ .vue ทันที

2. Component ที่ใช้บ่อยที่สุดในโปรเจกต์นี้

kendo-combobox / kendo-dropdownlist — dropdown เลือกค่า

<kendo-combobox
  :data-source="[
    { code: '001', desc: 'ยี่ห้อ A' },
    { code: '002', desc: 'ยี่ห้อ B' }
  ]"
  :data-text-field="'desc'"    <!-- field ที่จะโชว์ให้ user เห็น -->
  :data-value-field="'code'"   <!-- field ที่จะเป็นค่าจริงตอนเลือก -->
  :placeholder="'เลือกยี่ห้อ'"
></kendo-combobox>

kendo-numerictextbox — กรอกตัวเลข

<kendo-numerictextbox
  :max="100"
  :decimals="6"
  :format="'n6'"
  :spinners="true"
  :step="0.00001"
  @change="onChange"
></kendo-numerictextbox>

kendo-datepicker — เลือกวันที่

<kendo-datepicker
  v-model="birthDate"
  :format="'dd/MM/yyyy'"
  :min="minDate"
  :max="maxDate"
  @change="onChange"
></kendo-datepicker>

kendo-window — dialog/popup (โปรเจกต์ครอบไว้เป็น <kendo-window> อีกที)

โปรเจกต์นี้ไม่ได้ใช้ Window จาก Kendo ตรง ๆ แต่ครอบด้วย shared component ชื่อ BaseWindow.vue แล้ว register เป็น global component ชื่อ <kendo-window> (ดูบทที่ 08) 

<kendo-window ref="myWindow" :title="'ยืนยันการทำรายการ'" :width="400" :modal="true">
  <p>คุณต้องการบันทึกข้อมูลใช่หรือไม่?</p>
  <button @click="onConfirm">ยืนยัน</button>
</kendo-window>

<script>
methods: {
  openDialog() {
    this.$refs.myWindow.kendoWidget().center().open(); // เรียก Kendo widget API ตรง ๆ ผ่าน .kendoWidget()
  }
}
</script>

kendo-treeview — เมนู/โครงสร้างต้นไม้

ใช้เป็นเมนู sidebar หลักของทั้งระบบ (ดู MasterLayout.vue บทที่ 08)

<kendo-treeview
  :data-source="navDataSource"
  :template="'<span class=\'#= item.iconCls #\'></span><span>#= item.title #</span>'"
  :data-text-field="['name']"
  @select="onSelectMenu"
></kendo-treeview>

kendo-toolbar / kendo-toolbar-item — แถบปุ่มด้านบนของหน้าจอ

โปรเจกต์นี้ครอบไว้เป็น shared component MainActionToolbar อีกชั้น (ดูบทที่ 08) แต่ข้างในใช้ตัวนี้

<kendo-toolbar>
  <kendo-toolbar-item type="button" icon="save" text="บันทึก" @click="onSave"></kendo-toolbar-item>
  <kendo-toolbar-item type="button" icon="reset" text="เคลียร์" @click="onReset"></kendo-toolbar-item>
</kendo-toolbar>

3. Grid + DataSource — คู่หูที่ใช้เยอะที่สุดในระบบ

Kendo Grid แบบเก่าต้องคู่กับ kendo-datasource เพื่อจัดการ paging/filter/sort แบบฝั่ง server (โปรเจกต์นี้ครอบ pattern นี้ไว้เป็น shared component GridProxy.vue แล้ว — ปกติไม่ต้องเขียนเองตั้งแต่ศูนย์ ดูบทที่ 08) 

<kendo-datasource
  ref="ds"
  :transport="{
    read: { url: '/api/customer/search' }
  }"
  :schema-data="'content'"      <!-- key ใน response ที่เป็น array ข้อมูลจริง -->
  :schema-total="'total'"       <!-- key ใน response ที่บอกจำนวนทั้งหมด (สำหรับ paging) -->
  :server-filtering="true"
  :server-paging="true"
  :server-sorting="true"
  :page-size="10"
></kendo-datasource>

<kendo-grid
  :data-source="ds"
  :columns="[
    { field: 'code', title: 'รหัส' },
    { field: 'name', title: 'ชื่อ' }
  ]"
  :sortable="true"
  :pageable="true"
></kendo-grid>
ทางลัด: ใช้ GridProxy แทนการเขียน Grid+DataSource เอง
แทบทุกหน้าจอในโปรเจกต์นี้ที่ต้องการตารางข้อมูลแบบแบ่งหน้า จะ import GridProxy จาก faco.core@shared/Controls/Grid/GridProxy แล้วส่ง columns, url เข้าไปแทน — ไม่ต้องประกอบ datasource+grid เอง (ดูตัวอย่างเต็ม ๆ ในบทที่ 08)

4. kendo-upload — อัปโหลดไฟล์

<kendo-upload
  :async="{ saveUrl: uploadUrl, removeUrl: removeUrl, autoUpload: true }"
  @success="onSuccessUploadFile"
  @error="onErrorUploadFile"
  @remove="onRemoveFile"
></kendo-upload>

โปรเจกต์นี้มี global mixin method ชื่อ _onSuccessUploadFile, _onErrorUploadFile, _onRemoveFile ให้ใช้ได้ทุก component อยู่แล้ว (ประกาศไว้ใน CreateVueInstance.js) — ไม่ต้องเขียน handler อัปโหลดไฟล์เองใหม่ทุกครั้ง

5. เข้าถึง Kendo widget object ตรง ๆ เมื่อ component API ไม่พอ

บาง widget ต้องเรียก method พิเศษที่ไม่มีใน props/events ของ Vue wrapper ให้ใช้ this.$refs.xxx.kendoWidget()

this.$refs.myGrid.kendoWidget().dataSource.read();   // สั่ง refresh grid
this.$refs.myDate.kendoWidget().value(new Date());   // set ค่าตรง ๆ ผ่าน Kendo API

สรุปสิ่งที่ต้องจำ

  • Kendo component ถูก register เป็น global แล้วทั้งหมด ใช้ <kendo-xxx> ได้เลยไม่ต้อง import
  • มี 2 เจนเนอเรชันปนกัน: kendo-*-vue-wrapper (เก่า ใช้เยอะสุด) กับ @progress/kendo-vue-* (ใหม่)
  • Grid ต้องคู่กับ DataSource เสมอสำหรับข้อมูลแบบแบ่งหน้า — แต่ในทางปฏิบัติให้ใช้ GridProxy ที่ทีมทำไว้แทน
  • เข้าถึง Kendo widget API ตรง ๆ ผ่าน this.$refs.xxx.kendoWidget() เมื่อ Vue props/events ไม่พอ
