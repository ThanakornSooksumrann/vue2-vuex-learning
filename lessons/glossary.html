คำศัพท์เฉพาะ (Glossary)

รวมศัพท์เทคนิคทั้งหมดที่เจอในบทเรียน 01-09 อธิบายเป็นภาษาง่าย ๆ ไว้เปิดดูเวลาอ่านบทเรียนแล้วเจอคำที่ไม่คุ้น แต่ละคำมีป้าย บทที่ N บอกว่าเจอครั้งแรกในบทไหน กดลิงก์บทนั้นเพื่อย้อนไปดูบริบทเต็ม ๆ ได้

1. ศัพท์พื้นฐานของ Vue.js

คำศัพท์ความหมาย
Component ชิ้นส่วน UI ที่ใช้ซ้ำได้ มี template + logic ของตัวเอง เปรียบเหมือน "เลโก้" ที่เอามาต่อกันเป็นหน้าเว็บ บทที่ 01
Vue instance object ที่เกิดจาก new Vue({...}) คือตัวแอปหรือ component ที่กำลังทำงานอยู่จริงในหน่วยความจำ บทที่ 01
Options API วิธีเขียน Vue component แบบใส่ data/methods/computed/watch เป็น key ใน object เดียว (ตรงข้ามกับ Composition API ของ Vue 3 ที่เขียนแบบ function) — โปรเจกต์นี้ใช้ Options API ล้วน บทที่ 01
Template syntax ภาษาพิเศษที่เขียนใน <template> เช่น {{ }}, v-if, :src เพื่อบอก Vue ว่าจะ render HTML ยังไงจาก data บทที่ 01
Directive คำสั่งพิเศษที่ขึ้นต้นด้วย v- ใน template เช่น v-if, v-for, v-model, v-show, v-bind, v-on บทที่ 01
Reactivity (รีแอคทีฟ) ความสามารถหลักของ Vue: พอ data เปลี่ยนค่า หน้าจอ (DOM) จะอัปเดตให้อัตโนมัติ โดยไม่ต้องเขียนโค้ดสั่ง update เอง บทที่ 01
Two-way binding การผูกข้อมูลแบบ 2 ทาง — แก้ค่าใน input แล้ว data เปลี่ยนตาม, แก้ data แล้ว input ก็เปลี่ยนตาม ทำผ่าน v-model บทที่ 01
Computed property ค่าที่คำนวณจาก data ตัวอื่น และถูก "จำ" (cache) ไว้ จะคำนวณใหม่ก็ต่อเมื่อ data ที่มันอ้างอิงเปลี่ยนเท่านั้น บทที่ 01
Watcher (watch) ตัวดักฟังว่า data/computed ตัวไหนเปลี่ยนค่า แล้วสั่งทำ side-effect เช่น เรียก API หรือเขียน log บทที่ 01
Props ข้อมูลที่ parent component ส่ง "ลงมา" ให้ child component — child ห้ามแก้ props ตรง ๆ บทที่ 01
$emit / Event วิธีที่ child component ส่งข้อมูล/แจ้งเหตุการณ์ "ขึ้นไป" หา parent — parent ดักฟังด้วย @ชื่อ-event บทที่ 01
Slot "ช่องว่าง" ใน child component ที่ให้ parent ใส่เนื้อหา (HTML/component อื่น) เข้าไปแทนได้ ถ้าตั้งชื่อเรียกว่า named slot บทที่ 01
:key (ใน v-for) ค่าที่บอก Vue ว่าแต่ละ item ใน list ไหนคือ "ตัวเดิม" เวลาข้อมูลเปลี่ยน ช่วยให้ Vue อัปเดต DOM ได้แม่นยำและเร็วขึ้น ต้องใส่ทุกครั้งที่ใช้ v-for บทที่ 01
DOM Document Object Model คือโครงสร้าง HTML ที่เบราว์เซอร์ใช้แสดงผลหน้าเว็บจริง ๆ — Vue คอยจัดการ DOM แทนเราผ่าน reactivity บทที่ 01

2. ศัพท์เรื่อง Lifecycle & Asynchronous (การทำงานแบบรอผล)

คำศัพท์ความหมาย
Lifecycle hook ฟังก์ชันพิเศษที่ Vue เรียกให้เองตามช่วงอายุของ component เช่น created, mounted, beforeDestroy บทที่ 02
$refs วิธีอ้างอิงถึง DOM element หรือ component ลูกตรง ๆ จากโค้ด ใช้ได้ตั้งแต่ mounted เป็นต้นไปเท่านั้น (ก่อนหน้านั้นยังไม่มี DOM) บทที่ 02
Polling เทคนิคเรียก API ซ้ำ ๆ เป็นช่วงเวลา (เช่นทุก 5 วินาที) เพื่อเช็คข้อมูล/สถานะล่าสุด มักทำคู่กับ setInterval บทที่ 02
Memory leak บั๊กที่เกิดจากลืมเคลียร์ของ (เช่น setInterval, event listener) ตอน component ถูกทำลาย ทำให้โค้ดยังทำงานค้างอยู่เบื้องหลังทั้งที่ผู้ใช้ออกจากหน้าไปแล้ว บทที่ 02
Promise object ที่แทน "ผลลัพธ์ในอนาคต" ของงานที่ยังไม่เสร็จตอนนี้ (เช่นการเรียก API) มีสถานะ pending → resolved (สำเร็จ) หรือ rejected (ล้มเหลว) บทที่ 02
async / await syntax ที่ทำให้เขียนโค้ดที่ต้อง "รอผล" (asynchronous) ให้อ่านง่ายเหมือนโค้ดปกติ แทนที่จะต้องเขียน .then() ซ้อนกันหลายชั้น บทที่ 02
try / catch / finally โครงสร้างดักจับ error: try = โค้ดที่อาจพัง, catch = ทำเมื่อพัง, finally = ทำเสมอไม่ว่าจะสำเร็จหรือพัง (เช่นปิด loading) บทที่ 02

3. ศัพท์เรื่อง Vue Router

คำศัพท์ความหมาย
SPA (Single Page Application) เว็บแอปที่เปลี่ยน "หน้า" โดยไม่ reload browser ทั้งหน้า — ใช้ JavaScript สลับ component แทน บทที่ 03
Route การจับคู่ระหว่าง URL path กับ component ที่จะแสดง เช่น /customer → Customer.vue บทที่ 03
<router-view> "ช่อง" ในหน้า layout ที่ component ของ route ปัจจุบันจะไป render ใส่ บทที่ 03
<router-link> tag ที่ใช้แทน <a> เพื่อเปลี่ยนหน้าแบบ SPA (ไม่ reload browser) บทที่ 03
history mode / hash mode รูปแบบ URL ของ SPA — history mode ได้ URL สวย (/customer) แต่ต้องตั้งค่า server รองรับ, hash mode มี # คั่น (/#/customer) ใช้ได้แม้เปิดไฟล์ static ตรง ๆ บทที่ 03
Navigation guard ฟังก์ชันที่ดักทำงานก่อน/หลังเปลี่ยนหน้า เช่น beforeEach, afterEach ใช้เช็คสิทธิ์เข้าหน้า หรือทำ cleanup บทที่ 03
Dynamic params ส่วนของ path ที่เปลี่ยนได้ เช่น /customer/:id อ่านค่าได้จาก this.$route.params.id บทที่ 03
Query string ส่วนต่อท้าย URL หลัง ? เช่น ?mode=Inquiry อ่านค่าได้จาก this.$route.query.mode บทที่ 03
Lazy loading / Dynamic import การโหลด component เฉพาะตอนที่ผู้ใช้เข้า route นั้นจริง ๆ (เขียนด้วย () => import(...)) แทนที่จะโหลดทุกหน้าตั้งแต่แรก ทำให้แอปโหลดเร็วขึ้น บทที่ 03

4. ศัพท์เรื่อง Vuex (State Management)

คำศัพท์ความหมาย
State ข้อมูลกลางที่เก็บไว้ใน store ให้ทุก component เข้าถึงได้ (เหมือน data แต่ใช้ร่วมกันทั้งแอป) บทที่ 04
Getter เหมือน computed แต่ใช้ร่วมกันได้ทั้งแอป อ่านค่าที่คำนวณจาก state บทที่ 04
Mutation ฟังก์ชันเดียวที่อนุญาตให้แก้ state ได้ตรง ๆ ต้องเป็น synchronous เท่านั้น (ห้ามมี await/API เรียกอยู่ข้างใน) บทที่ 04
Action ฟังก์ชันที่ทำ logic/เรียก API แบบ async ได้ แล้วค่อย commit mutation เพื่อเปลี่ยน state จริง บทที่ 04
commit คำสั่งเรียก mutation เช่น this.$store.commit('INCREMENT') บทที่ 04
dispatch คำสั่งเรียก action เช่น this.$store.dispatch('fetchProfile') — component ควรเรียก action ผ่าน dispatch เสมอเวลามี async บทที่ 04
Module / namespaced การแยก store ก้อนใหญ่เป็นส่วนย่อยตามฟีเจอร์ namespaced: true ทำให้ต้องเรียกผ่าน path เต็ม (เช่น user/fetchProfile) กันชื่อชนกับ module อื่น บทที่ 04
CRUD ย่อจาก Create-Read-Update-Delete คือ 4 การกระทำพื้นฐานกับข้อมูล (เพิ่ม/อ่าน/แก้/ลบ) บทที่ 04
vuex-pathify library เสริมของ Vuex ที่ auto-generate mutation/action พื้นฐานให้ และมี syntax สั้น ๆ อย่าง sync(), $store.get(), $store.set() — โปรเจกต์นี้ใช้ตัวนี้แทน mapState/mapActions มาตรฐาน บทที่ 04
sync() helper ของ pathify ที่ผูก state เป็น computed (อ่าน+เขียนได้) ในบรรทัดเดียว เช่น ...sync('user/profile') บทที่ 04
เครื่องหมาย ! ต่อท้าย (pathify) convention ของ pathify ที่บอกว่ากำลังเรียก action (dispatch) ไม่ใช่ set ค่า state ตรง ๆ เช่น $store.set('Module/doSomething!', payload) บทที่ 04

5. ศัพท์เรื่อง Kendo UI / UI Library ทั่วไป

คำศัพท์ความหมาย
Widget ชื่อเรียก UI component ของ Kendo (เช่น Grid widget, DatePicker widget) — มาจากยุค jQuery ก่อนที่จะมี Vue wrapper ครอบ บทที่ 05
Wrapper โค้ดที่ครอบของเดิม (เช่น Kendo widget แบบ jQuery) ให้ใช้งานแบบ Vue component ได้ (ผ่าน props/events ปกติของ Vue) บทที่ 05
Installer (เช่น GridInstaller) plugin ของ Kendo ที่เรียกผ่าน Vue.use(...) เพื่อลงทะเบียน component ชุดนั้นให้ใช้ได้ทั้งแอป บทที่ 05
DataSource ตัวกลางของ Kendo ที่จัดการการดึงข้อมูล (paging/filter/sort) ให้ Grid หรือ widget อื่น ๆ บทที่ 05
Server-side paging / filtering / sorting การแบ่งหน้า/กรอง/เรียงข้อมูล ทำที่ฝั่ง server (backend) แล้วส่งกลับมาแค่หน้าที่ต้องการ แทนที่จะโหลดข้อมูลทั้งหมดมากรองที่ browser บทที่ 05
kendoWidget() method ที่เรียกผ่าน this.$refs.xxx.kendoWidget() เพื่อเข้าถึง Kendo widget object ตรง ๆ เวลา props/events ของ Vue wrapper ไม่พอ บทที่ 05

6. ศัพท์เฉพาะของโปรเจกต์ KL.Core.UI

คำศัพท์ความหมาย
Entry point ไฟล์แรกที่ webpack เริ่มรันตอนโหลดแอป ของโปรเจกต์นี้คือ src/main.js บทที่ 06
Mixin ก้อน logic (data/methods/lifecycle) ที่ผสมเข้ากับทุก component โดยอัตโนมัติ — โปรเจกต์นี้ใช้ Vue.mixin({...}) ใน CreateVueInstance.js เพื่อแจก method กลาง (เช่น upload, hub connection) ให้ทุก component ใช้ได้เลย บทที่ 06
Prototype property การเติม property เข้าไปที่ต้นแบบของ Vue (Vue.prototype) ทำให้ทุก component เรียกใช้ได้ผ่าน this.xxx เช่น this.$utils บทที่ 06
Singleton รูปแบบที่มี instance เดียวใช้ร่วมกันทั้งระบบ ไม่สร้างใหม่ซ้ำ ๆ เช่น $utils ที่เป็น utility ตัวเดียวกันไม่ว่าจะเรียกจาก component ไหน บทที่ 06
Polyfill โค้ดที่เติมความสามารถของ JavaScript รุ่นใหม่ให้ browser รุ่นเก่าใช้งานได้ (โปรเจกต์นี้ import babel-polyfill ไว้ที่บรรทัดแรกของ main.js) บทที่ 06
Webpack alias ชื่อย่อที่ตั้งไว้ใน webpack config แทน path ยาว ๆ เช่น faco.core@shared แทน src/app/shared ทำให้ import สั้นและไม่พังตอนย้ายไฟล์ บทที่ 06
vmMaster / Multi-instance รูปแบบที่ Vue instance หนึ่งถูกฝังอยู่ในอีก instance หนึ่ง (แอปลูกในแอปแม่) — เช็คได้จาก this.$root.vmMaster บทที่ 06
$configCore ตัวแปร config ระดับ global (ไม่ใช่ของ Vue) เก็บค่าต่อ environment เช่น URL ของแต่ละ API, ชื่อแอป บทที่ 06
SCOPE_MODULE ตัวแปร environment ที่ใช้ตอนรัน dev server เพื่อจำกัดว่าจะ build เฉพาะ module ไหน (เช่น Finance, HPAdmin) ลดเวลา build ตอน dev บทที่ 06
SignalR / Hub connection เทคโนโลยี real-time ของ Microsoft ใช้เปิดการเชื่อมต่อค้างไว้ (WebSocket) ระหว่าง browser กับ server เพื่อรับข้อมูล/แจ้งเตือนแบบสด ๆ โดยไม่ต้อง refresh บทที่ 06
require.context ฟังก์ชันพิเศษของ Webpack ที่สแกนหาไฟล์ทั้งหมดในโฟลเดอร์ตาม pattern ที่กำหนด แล้ว import ให้อัตโนมัติ — กลไกหลักที่ทำให้ route และ Vuex module ของโปรเจกต์นี้ "auto-register" เอง บทที่ 07
Auto-register / Auto-generate คำเรียกกลไกที่ route หรือ Vuex module ถูกสร้างขึ้นเองจากตำแหน่งไฟล์ในโฟลเดอร์ โดยไม่ต้องเขียนโค้ดเพิ่ม/แก้ที่ไหนอีก บทที่ 07
make.getters/mutations/actions helper ของ vuex-pathify ที่สร้าง getter/mutation/action พื้นฐานให้ทุก property ใน state โดยอัตโนมัติ ไม่ต้องเขียนเอง บทที่ 07
Interceptor โค้ดที่ "ดัก" ทุก request/response ของ axios ก่อนส่ง/หลังได้รับ เพื่อทำอะไรบางอย่างอัตโนมัติ เช่น แนบ token, เปิด/ปิด loading modal, จัดการ error กลาง ๆ บทที่ 08
Repository pattern รูปแบบที่รวมการเรียก API ทั้งหมดไว้ที่ไฟล์เดียว (repository.js) แทนที่จะกระจาย import axios ไปทั่วโปรเจกต์ บทที่ 08
Cell component / Cell editor component เล็ก ๆ ที่ใช้เป็น template ของแต่ละคอลัมน์ใน Grid เมื่อคอลัมน์นั้นต้องการ input พิเศษ (ไม่ใช่ text ธรรมดา) เช่น dropdown, date picker บทที่ 08
เจอศัพท์ที่ไม่มีในตารางนี้?
ถ้าเจอคำศัพท์ในโค้ดจริงที่ไม่ได้อธิบายไว้ที่นี่ ลองเปิดบทเรียนที่เกี่ยวข้องดูก่อน (เช่นคำเกี่ยวกับ Grid ให้ดูบทที่ 05/08) หรือค้นหาชื่อ package นั้นใน package.json ของโปรเจกต์เพื่อดูว่าเป็น library อะไร แล้วค้นเพิ่มได้จากเอกสารทางการของ library นั้น ๆ
