Shared Components & Layout ที่ทีมทำไว้แล้ว

กฎสำคัญที่สุดของบทนี้: ก่อนจะเขียนอะไรใหม่ ให้เช็คก่อนว่ามี shared component ทำไว้ให้แล้วหรือยัง ทั้งหมดอยู่ใต้ src/app/shared/ เข้าถึงผ่าน alias faco.core@shared/...

1. MasterLayout.vue — เปลือกหลักของทั้งแอป

ไฟล์จริง: src/app/shared/Layout/MasterLayout.vue — component นี้ถูกแสดงโดย App.vue หลัง login สำเร็จ โครงสร้างคือ:

<b-app>
  <header>                     <!-- โลโก้, ชื่อหน้า/breadcrumb, ปุ่ม logout, สลับภาษา, ชื่อ user -->
  <aside>
    <kendo-treeview :data-source="navDataSource" @select="onSelectMenu" />  <!-- เมนู sidebar -->
  </aside>
  <section class="main-content">
    <router-view :key="fullPath" class="view"></router-view>   <!-- หน้าจอปัจจุบันมาโผล่ตรงนี้ -->
  </section>
</b-app>
ทำไม router-view ถึงมี :key="fullPath"?
ปกติถ้าเปลี่ยนแค่ query string (เช่น ?mode=Inquiry → ?mode=Edit) โดย path เดิม Vue Router จะไม่ re-create component ใหม่ (แค่ update props) การใส่ :key="fullPath" (path + query) บังคับให้ Vue เห็นว่าเป็น component คนละตัวทุกครั้งที่ query เปลี่ยน จึงจะเรียก created/mounted ใหม่เสมอ — สำคัญมากเพราะโปรเจกต์นี้พึ่งพา mode/modeFlag/modePage จาก query string เพื่อ reset สถานะของหน้าจอ

เวลาสร้างหน้าจอใหม่ ไม่ต้องไปยุ่งกับ MasterLayout เลย — แค่สร้างไฟล์ view ให้ถูกที่ (บทที่ 07/09) มันจะไปโผล่ใน router-view ให้เอง

2. Grid ตระกูล GridProxy / GridDisplay — ไม่ต้องประกอบ Kendo Grid เอง

ไฟล์: src/app/shared/Controls/Grid/GridProxy.vue — ครอบ kendo-datasource + GridDisplay ให้แล้ว รองรับ server-side paging/filter/sort

<GridProxy
  ref="myGrid"
  :columns="[
    { field: 'code', title: 'รหัสลูกค้า' },
    { field: 'name', title: 'ชื่อลูกค้า' }
  ]"
  :url="'/api/customer/search'"
  :parameters="{ status: 'active' }"
  :height="400"
  @onRowClick="onRowClick"
>
  <template #btnOptionR><button>Export</button></template>   <!-- ปุ่มเสริมมุมขวาบน -->
</GridProxy>
Componentใช้เมื่อไหร่
GridProxyต้องการตารางที่ดึงข้อมูลจาก API เอง พร้อม paging/filter/sort ฝั่ง server (ใช้บ่อยที่สุด)
GridDisplayแสดงตารางจาก data ที่มีอยู่แล้วใน component (ไม่ต้องยิง API เอง — GridProxy ใช้ตัวนี้อยู่ข้างในอีกที)
GridLazyLoadตารางแบบ virtual scroll (ข้อมูลเยอะมาก โหลดเพิ่มตอน scroll)

3. MainActionToolbar — แถบปุ่มมาตรฐานด้านบนหน้าจอ

ไฟล์: src/app/shared/Controls/Toolbar/MainActionToolbar.vue — ให้ปุ่ม save/reset มาตรฐานพร้อมข้อความแปลภาษาให้แล้ว เพิ่มปุ่มพิเศษผ่าน default slot ได้

<MainActionToolbar :actions="['reset']" @onReset="onReset">
  <kendo-toolbar-item type="button" icon="search" :text="$t('FF-BT-000003')" @click="onSearch" />
</MainActionToolbar>

4. FormGroup + Vuelidate — validate ฟอร์มแบบมาตรฐาน

ไฟล์: src/app/shared/Controls/Form/FormGroup.vue — ถูก register เป็น plugin ของ vuelidate-error-extractor ไว้แล้วในระดับแอป ทำให้ error message ของ Vuelidate โชว์เป็น tooltip ที่หน้าตาเหมือนกันทั้งระบบโดยอัตโนมัติ ไม่ต้องเขียน error display เอง

5. Modal / Dialog มาตรฐาน — เรียกผ่าน App.vue ได้จากทุกที่

Componentไฟล์ใช้ทำอะไร
AlertMessageshared/Controls/Modal/AlertMessage.vueแจ้งเตือน/ยืนยัน (title, message, status code)
LoadingModalshared/Controls/Modal/LoadingModal.vueหน้าจอ loading ระหว่างรอ API (ถูกเปิด/ปิดอัตโนมัติจาก interceptor ของ axios)
SearchModalshared/Controls/Modal/SearchModal.vuepopup ค้นหา/เลือกข้อมูล (lookup)
BaseWindow (<kendo-window>)shared/BaseComponents/BaseWindow.vueครอบ Kendo Window ไว้อีกชั้น ส่ง slot/listener ทะลุผ่านให้หมด
 
// เรียก alert จาก component ไหนก็ได้ ผ่าน App.vue (root)
this.$root.App.alertMessage("แจ้งเตือน", "บันทึกไม่สำเร็จ กรุณาลองใหม่", 400);

// เปิด/ปิด kendo-window ของตัวเอง
this.$refs.myWindow.kendoWidget().center().open();

6. Cell components — ตัว editor ในแต่ละคอลัมน์ของ Grid

อยู่ใต้ src/app/shared/Controls/Cell/ ใช้เป็น custom cell template ของ Kendo Grid เมื่อ column ต้องการ input พิเศษ (ไม่ใช่แค่ text ธรรมดา)

Componentใช้กับคอลัมน์แบบไหน
CellTextboxกรอกข้อความ
CellNumericตัวเลข (มี min/max/decimal/spinners)
CellDatePickerเลือกวันที่
CellDropDownStatusเลือกสถานะจาก dropdown
CellCheckboxติ๊กถูก
CellHyperLinkลิงก์ที่คลิกแล้วเปิดหน้าอื่น/emit event
CellUploadFileอัปโหลดไฟล์ในแถวของ grid

7. repository.js — จุดเดียวที่เรียก API ทั้งระบบผ่าน

ไฟล์: src/app/repositories/repository.js (import ผ่าน alias faco.core@repository) — ครอบ axios ไว้ให้แยกเป็นตัวแปรต่อ API

import axios from "faco.core@repository";

// axios.<ชื่อ api ตาม config> — แต่ละตัวมี baseURL, interceptor แนบ token ให้อัตโนมัติ
axios.financeApi.get('/Customer/Search', { params });
axios.systemConfigApi.get('/System/SystemDate');
axios.reportDataApi.post('/Export/Excel', payload);
ทำไมต้องเรียกผ่าน faco.core@repository เสมอ
Interceptor ใน repository.js จัดการให้อัตโนมัติ 3 อย่าง: แนบ token/clientId ทุก request, เปิด/ปิด LoadingModal อัตโนมัติ (ยกเว้นตั้ง __hideLoading__: true), และดักจัดการ error/401 กลาง ๆ ที่เดียว ถ้า import axios ตรง ๆ เองจะไม่ได้พฤติกรรมพวกนี้

สรุปสิ่งที่ต้องจำ

  • ไม่ต้องแตะ MasterLayout เอง — router-view จะแสดงหน้าจอที่สร้างให้อัตโนมัติ
  • ต้องการตาราง → ใช้ GridProxy ก่อนเสมอ อย่าประกอบ kendo-datasource+kendo-grid เอง
  • ต้องการปุ่ม save/reset มาตรฐาน → MainActionToolbar
  • ต้องการ validate ฟอร์ม → FormGroup + Vuelidate
  • ต้องการแจ้งเตือน/popup → AlertMessage, LoadingModal, SearchModal, kendo-window
  • เรียก API เสมอผ่าน faco.core@repository ไม่ import axios ตรง ๆ
