Shared Component / Store / Utility ทั้งหมด (Full Reference)

บทที่ 08 พาดูแค่ตัวหลัก ๆ ที่ใช้บ่อยที่สุด แต่โปรเจกต์นี้สร้างของกลางไว้เยอะกว่านั้นมาก หน้านี้รวบรวมทุกตัวที่อยู่ใต้ src/app/shared/ (Vue component), src/app/store/shared/ (Vuex module), และทุก method ของ this.$utils — ใช้เป็น reference เปิดหาเวลาจะเขียนฟีเจอร์ใหม่ ก่อนเขียนเองให้เช็คตารางพวกนี้ก่อนเสมอว่ามีของสำเร็จรูปอยู่แล้วหรือไม่

วิธีใช้หน้านี้
กด Ctrl/Cmd + F ค้นหาชื่อสิ่งที่ต้องการ (เช่น "upload", "date", "lookup") แทนการไล่อ่านทั้งหมด แต่ละตัวมี path ไฟล์จริงกำกับไว้ เปิดไฟล์นั้นดู prop/option เต็ม ๆ ได้เสมอเพราะบางตัวมี option เยอะกว่าที่สรุปไว้ที่นี่

1. Shared Vue Components (src/app/shared/)

1.1 Cell/ — ตัว editor/แสดงผลของแต่ละคอลัมน์ใน Grid

ใช้เป็น custom cell template ผ่าน columns[].cell หรือ editor ของ GridDisplay/GridProxy ทุกตัวรับ props field, dataItem, options เหมือนกัน และ emit itemchange({dataItem, field, value}) เมื่อมีการแก้ไข

Componentใช้ทำอะไรOptions/Props เด่น
CellTextboxช่องกรอกข้อความ 1 บรรทัดmaxLength, fnReadonly, fnColor
CellTextareaช่องกรอกข้อความหลายบรรทัดmaxLength, fnReadonly
CellNumericช่องกรอกตัวเลข (Kendo spinner)min, max, decimal, spinners, fnReadonly
CellDisplayNumberแสดงตัวเลขอย่างเดียว จัดรูปแบบให้ (อ่านอย่างเดียว ไม่แก้ไข)decimal, fnColor, fnRenderer
CellDatePickerเลือกวันที่ในเซลล์fnMin, fnMax (ฟังก์ชันคำนวณช่วงวันที่ได้)
CellDisplayDatePickerแสดงวันที่อย่างเดียว จัดรูปแบบไทย/อังกฤษให้format
CellCheckboxติ๊กถูก/ผิดในเซลล์checkValue, uncheckValue, align

CellDropDownStatusเลือกสถานะจาก dropdown ในเซลล์มักผูกกับ list สถานะคงที่
CellLookupค้นหา/เลือกข้อมูล master (เช่น รหัสลูกค้า) ในเซลล์ — ใช้ <Lookup> ข้างในอีกทีcodeField, displayField, columns, transport, mode, widthCode, fnHideComponent, parameter
CellHyperLinkลิงก์แบบ static ที่คลิกแล้ว emit eventemit onClickCellLink
CellDynamicHyperLinkลิงก์ที่ปลายทาง route เปลี่ยนได้ตาม dataItememit onClickCellLink
CellButtonปุ่ม action ในแถว (เช่น ปุ่มดู/แก้ไข)text, icon, width, align; emit onClickCellButton(event, dataItem)

CellInputTimeกรอกเวลา (ชั่วโมง:นาที) แบบ masked inputใช้คู่กับ field เวลาแยกจาก date
CellTextSplitแสดง/แยกข้อความที่เป็น pattern พิเศษในเซลล์ใช้เมื่อ field เดียวต้องแยกแสดงหลายส่วน

ตัวอย่างการใช้งาน: นิยาม columns ของ Grid โดยผูก Cell component ให้แต่ละคอลัมน์ตามชนิดข้อมูล แล้วรับการแก้ไขทั้งหมดที่ event เดียว

data() {
  return {
    columns: [
      { field: "code", title: "รหัส", width: 100 },
      { field: "name", title: "ชื่อลูกค้า", editor: "CellTextbox", options: { maxLength: 100 } },
      { field: "amount", title: "ยอดเงิน", editor: "CellNumeric", options: { min: 0, decimal: 2 } },
      { field: "dueDate", title: "วันครบกำหนด", editor: "CellDatePicker" },
      { field: "isActive", title: "ใช้งาน", editor: "CellCheckbox" },
      { field: "customerCode", title: "ลูกค้า", editor: "CellLookup", options: {
          codeField: "code", displayField: "name",
          transport: { read: { url: "/Customer/Search" } }
        }
      },
      { field: "file", title: "ไฟล์แนบ", editor: "CellUploadFile" },
      { field: "action", title: "", editor: "CellButton", options: { icon: "edit", text: "แก้ไข" } }
    ]
  };
},
methods: {
  // ทุก Cell component ที่แก้ไขได้จะยิง event นี้เหมือนกันหมด
  onItemChange({ dataItem, field, value }) {
    this.$set(dataItem, field, value);   // อัปเดตค่าลง dataItem ของแถวนั้นโดยตรง
  },
  onClickCellButton(event, dataItem) {
    this.$utils.routerPush(`/Customer/Edit?code=${dataItem.customerCode}`);
  }
}

1.2 Grid/ — Grid หลักและตัวช่วย

Componentใช้ทำอะไรProps/Events เด่น
GridProxyGrid ที่ดึงข้อมูลจาก API เอง พร้อม server-side paging/filter/sort — ตัวที่ใช้บ่อยสุดprops: columns, url, parameters, height; events: onRowClick, onClickCellLink, dataStateChange
GridDisplayGrid ที่รับ data มาแสดงตรง ๆ (ไม่ยิง API เอง) — GridProxy ใช้ตัวนี้ข้างในอีกทีprops: columns, dataItems, totalItems, editField, enableVirtual, detail; methods: clear(), getAllData(), cancelChanges()
GridLazyLoadโหลดข้อมูลจำนวนมากแบบเป็นก้อน (virtual scroll) เหมาะกับข้อมูลหลักหมื่น-แสนแถวprops: url, sizeLoad, startSizeLoad, fetchMethod; methods: fetch(), getDataChange(), importData(); คู่กับ store Controls_Grid_GridLazyLoad
GridNativeตัวห่อ Kendo Grid ดิบ ๆ (ไม่ค่อยเรียกตรง ๆ — ตัวอื่นข้างบนห่อไว้ให้แล้ว)ใช้เมื่อต้อง customize Grid ลึกจริง ๆ เท่านั้น
Filters/ColumnMenuเมนู filter ต่อคอลัมน์ (ตัวกรองแบบ Excel)ใช้ผูกกับ columnMenu ของ Grid
Filters/FilterItems/FilterDateMaskedช่อง filter วันที่แบบ masked inputใช้เป็น custom filter cell ของคอลัมน์วันที่

ตัวอย่างการใช้งาน: GridProxy พร้อมปุ่มเสริมผ่าน slot และดักคลิกแถว

1.3 Modal/ — popup และ dialog

Componentใช้ทำอะไรMethods/Props เด่น
AlertMessageแจ้งเตือน/ยืนยัน แสดง title+message พร้อมสีตาม status (400 เตือน, 500 error, 401 unauthorized)open(title, message, status), onClickOk(), onClickCancel()
LoadingModalหน้าจอ loading พร้อมข้อความสถานะ — เปิด/ปิดอัตโนมัติจาก interceptor ของ axios ผ่าน store Controls_Modal_LoadingModalopen(), close()
SearchModalpopup ค้นหา/เลือกข้อมูล master ใช้คู่กับ Lookup/CellLookupopen(), initialConfig(cfg), onSelectItem(), reset()
PopupDynamicเปิด component ใด ๆ (ตาม route path) เป็น popup แบบ dynamic โดยไม่ต้องรู้จัก component นั้นล่วงหน้าprops: value (v-model), path, params, width, title

ตัวอย่างการใช้งาน:

<template>
  <div>
    <AlertMessage ref="alertMessage" />
    <LoadingModal ref="loadingModal" />   <!-- ปกติ App.vue มีให้อยู่แล้วระดับแอป ไม่ต้องประกาศเองในทุกหน้าจอ -->
    <SearchModal ref="searchModal" />
    <PopupDynamic v-model="showPopup" :path="'/Customer/QuickView'" :params="{ code: selectedCode }" title="ดูข้อมูลลูกค้า" />
  </div>
</template>

<script>
methods: {
  onDeleteClick() {
    // เปิด modal ยืนยัน — ใช้ callback fnOnYes/fnOnNo แทนการ await
    this.$refs.alertMessage.fnOnYes = () => { this.doDelete(); };
    this.$refs.alertMessage.open("ยืนยันการลบ", "ต้องการลบรายการนี้ใช่หรือไม่?", 400);
  },
  openLookup() {
    this.$refs.searchModal.initialConfig({
      columns: this.lookupColumns,
      url: "/Customer/Search",
      onSelect: (dataItem) => { this.selectedCode = dataItem.code; }
    });
    this.$refs.searchModal.open();
  },
  showQuickView(code) {
    this.selectedCode = code;
    this.showPopup = true;   // v-model ของ PopupDynamic เปิด/ปิด popup
  }
}
</script>
ไม่ต้องเรียก LoadingModal เอง
LoadingModal ถูกเปิด/ปิดอัตโนมัติจาก axios interceptor ทุกครั้งที่มี request/response (ผ่าน store Controls_Modal_LoadingModal) — ไม่ต้อง .open()/.close() เองในโค้ดหน้าจอทั่วไป

1.4 Input/ — input พิเศษที่ใช้ทั่วทั้งฟอร์ม

Componentใช้ทำอะไรProps/Events เด่น
Lookupช่องกรอกรหัส+ชื่อ พร้อมปุ่มเปิด SearchModal ค้นหา (ใช้แยกจาก Grid ได้ เช่นในฟอร์ม)props: value, columns, transport, mode ('both'/'codeOnly'/'displayOnly'), codeField, displayField; emit change(value, dataItem, source)
Triggerปุ่ม/ไอคอนติดท้าย input (ใช้เป็น building block ให้ Lookup)props: icon, disabledTrigger; slot: front, back; emit clickTrigger()

ตัวอย่างการใช้งาน: ช่องเลือกรหัสลูกค้าในฟอร์ม (นอก Grid)

<Lookup
  v-model="customerCode"
  :columns="[
    { field: 'code', title: 'รหัส' },
    { field: 'name', title: 'ชื่อลูกค้า' }
  ]"
  :transport="{ read: { url: '/Customer/Search' } }"
  :code-field="'code'"
  :display-field="'name'"
  :mode="'both'"
  @change="onCustomerChange"
/>

<script>
methods: {
  onCustomerChange(value, dataItem, source) {
    // dataItem = ข้อมูลเต็มของแถวที่เลือก, source บอกว่าเลือกจาก popup หรือพิมพ์เอง
    this.customerName = dataItem ? dataItem.name : "";
  }
}
</script>

<script>
import { required, maxLength } from "vuelidate/lib/validators";

export default {
  data() {
    return { form: { name: "" } };
  },
  validations: {
    form: {
      name: { required, maxLength: maxLength(100) }

  methods: {
    onSave() {
      this.$v.$touch();                 // สั่งเช็ค validate ทั้งฟอร์ม — ถ้าไม่ผ่าน FormGroup จะโชว์ tooltip error ให้เอง
      if (this.$v.$invalid) return;
      // ... ส่ง API ต่อ
    },
    onApprove() { /* ... */ },
    onReset() { this.$v.$reset(); }
  }
};
</script>

ตัวอย่างการใช้งาน: อ่าน/เขียน store กลางเหล่านี้ผ่าน vuex-pathify เหมือน module อื่น ๆ ทุกประการ (ดูบทที่ 07)

import { sync } from "vuex-pathify";

export default {
  computed: {
    ...sync("Layout_MasterLayout/*")   // ได้ this.titleName, this.subTitleName ใช้ตรง ๆ
  },
  mounted() {
    // ตั้งชื่อหัวข้อ/breadcrumb ของหน้าจอ — ทุกหน้าจอทำแบบนี้เหมือนกันหมด
    this.$store.set("Layout_MasterLayout/setTitle", {
      titleName: "ค้นหาลูกค้า",
      subTitleName: "Demo > ค้นหาลูกค้า"
    });

    // เช็คว่า auth เสร็จหรือยังก่อนเรียก API
    const completedAuth = this.$store.get("Layout_Main/completedAuth");
    if (completedAuth) {
      this.loadInitialData();
    }

    // ดูว่ามี loading ค้างอยู่กี่ request (ปกติไม่ต้องยุ่งเอง เพราะ interceptor จัดการให้)
    const loadingCount = this.$store.get("Layout_Loading/loading");
  }
};

3. Shared Utilities (this.$utils / this.$utilsCore)

ตัวนี้เป็น singleton object ผูกไว้กับทุก component แล้ว (ดูบทที่ 06) เรียกได้ทันทีที่ this.$utils.ชื่อฟังก์ชัน(...) ต่อไปนี้คือ method เกือบทั้งหมดที่มีอยู่ แบ่งตามหมวดการใช้งาน

3.1 ภาษา & Routing

Methodทำอะไร
getLang()คืนภาษาปัจจุบัน ('th' เป็นค่าเริ่มต้น)
setLang(lang, callback)เปลี่ยนภาษา บันทึกลง storage และอัปเดต vue-i18n locale
routerPush(path)เปลี่ยนหน้า — ถ้าอยู่ใน embed mode (มี vmMaster) จะส่งต่อให้แอปแม่จัดการแทน
getRoutePath() / getRoutePathNoParam()คืน path ปัจจุบัน (แบบมี/ไม่มี query string)
routeFromModule(path, lv)ตัดชื่อ module ออกจาก path เช่น /Finance/xxx → Finance
 
this.$utils.routerPush(`/Customer/Detail?code=${code}`);
const currentPath = this.$utils.getRoutePathNoParam();     // path ไม่มี query
const moduleName = this.$utils.routeFromModule(currentPath); // เช่น "Finance"

3.2 Storage & Session

Methodทำอะไร
getStorage(key) / setStorage(key, value) / removeStorage(key) / clearStorage()อ่าน/เขียน/ลบค่าที่เก็บไว้ฝั่ง client (แทน localStorage/sessionStorage ตรง ๆ)
getAuthStorage()คืนข้อมูล user ที่ login (token, userId, nameDisplay, role ฯลฯ) ทั้งก้อน
removeAuthStorage()ล้างข้อมูล login ทั้งหมด (ใช้ตอน logout)
getAppUrl()คืน base URL ของแอป (รวม appName)
 
this.$utils.setStorage("lastViewedCustomer", customerCode);
const lastCode = this.$utils.getStorage("lastViewedCustomer");

const authInfo = this.$utils.getAuthStorage();
console.log(authInfo.nameDisplay, authInfo.role);

3.3 System & Authentication

Methodทำอะไร
initialSystem()ดึงข้อมูล user เริ่มต้นตอนเปิดแอป (cache ไว้ถ้ามีแล้ว)
actionSignOut()ออกจากระบบ: ล้าง auth storage, cookies, ปิด hub connections, reset store flags
actionSignOutAzure()เหมือน actionSignOut() แต่ redirect ไปหน้า logout ของ Azure AD ด้วย
fetchSystemDate() / getSystemDate() / setCacheSystemDate(date)ดึง/อ่าน/ตั้งค่า วันที่ของ server (ไม่ใช้วันที่เครื่อง client เพื่อกันเวลาเพี้ยน)
clearCookies()ลบ cookie ทั้งหมดของ browser
registerKey() / decryptDataKey()ตรวจสอบ/ถอดรหัส security key หลัง OAuth handshake (ใช้ภายในระบบ auth เอง ไม่ค่อยเรียกตรง ๆ จากหน้าจอทั่วไป)

3.4 แปลง/จัดการข้อมูล (Data transform)

Methodทำอะไร
clone(data)deep clone object/array (กัน reference ชนกัน)
filterClone(val, fnFilter)clone แล้วกรองด้วยฟังก์ชันที่ส่งเข้าไป
trimJSON(json)ตัดช่องว่างหน้า-หลังของทุก string ใน object/array แบบ recursive ก่อนส่ง API
cloneGridData(data)clone array ของ grid แล้วลบ field ภายใน (_uuid_, _ID, _editField_, flagSave) ออกก่อนส่ง API จริง
cloneMatchProp(data, dataStructure)clone เฉพาะ property ที่มีอยู่ใน dataStructure ต้นแบบ
objectAssignMatchProp(dataStructure, ...updates)merge object โดยรับเฉพาะ key ที่มีอยู่ใน structure ต้นแบบเท่านั้น (กัน key แปลกปลอมหลุดเข้ามา)
updateGridData(dataFrom, dataTo)merge แถว grid โดยรักษาสถานะ flagSave ('I' insert / 'U' update / 'D' delete) ให้ถูกต้อง
pick(obj, prop1, prop2, ...)ดึงเฉพาะ property ที่ระบุออกมาเป็น object ใหม่
hookState(object, value)ใส่ค่าลง object พร้อมแปะ _id_ (uuid) ให้อัตโนมัติ
 
// สถานการณ์จริง: ก่อนส่ง API ให้ trim ข้อความ + ตัด field ภายในของ grid ออกก่อน
onSave() {
  const payload = this.$utils.trimJSON(this.$utils.clone(this.form));
  const gridDataToSave = this.$utils.cloneGridData(this.gridItems); // ลบ _uuid_, _editField_, flagSave-ภายในออก

  axios.financeApi.post("/Customer/Save", { ...payload, items: gridDataToSave })
    .then(() => this.$utils.addNotify("บันทึกสำเร็จ", 200));
}

3.11 ระบบภายใน / Security (รู้จักไว้ ไม่ค่อยเรียกเองตรง ๆ)
