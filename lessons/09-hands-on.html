ลงมือทำจริง: สร้างหน้าจอใหม่ 1 หน้าในโปรเจกต์นี้

รวมทุกอย่างจากบทที่ 01-08 มาเป็น checklist ที่ทำตามได้จริง สำหรับสร้างหน้าจอ CRUD/ค้นหาข้อมูลง่าย ๆ 1 หน้าใน KL.Core.UI ใช้ตัวอย่างสมมติ: หน้าจอ "ค้นหาลูกค้า" (SearchCustomer) ใน module Demo

ก่อนเริ่ม: ดูของตัวอย่างที่มีอยู่แล้วก่อนเสมอ
โฟลเดอร์ src/app/views/Demo/ (เช่น Demo2.vue, TodoList.vue, CustomerDemo.vue) และ src/app/store/views/Demo/Stores/ คือหน้าจอตัวอย่างที่ทีมทำไว้สำหรับฝึกเรียนรู้ pattern โดยเฉพาะ เปิดไฟล์พวกนี้เทียบไปพร้อมกับทำ checklist ด้านล่างได้เลย

ขั้นตอนที่ 1 — สร้างไฟล์หน้าจอ (View)

สร้างไฟล์ src/app/views/Demo/SearchCustomer/Index.vue

ทำไมต้องชื่อ Index.vue ในโฟลเดอร์แยก แทนที่จะเป็น SearchCustomer.vue ตรง ๆ
ทั้งสองแบบใช้ได้ (route จะสร้างจาก path เสมอ) แต่ทีมนิยมแยกเป็นโฟลเดอร์ + Index.vue เมื่อหน้าจอมีหลาย component ย่อย (filter, detail, popup) เพื่อเก็บไฟล์ที่เกี่ยวข้องกันไว้ในโฟลเดอร์เดียว เช่น SearchCustomer/Components/SearchCustomerFilter.vue 
<template>
  <div>
    <MainActionToolbar :actions="['reset']" @onReset="onReset">
      <kendo-toolbar-item type="button" icon="search" :text="$t('FF-BT-000003')" @click="onSearch" />
    </MainActionToolbar>

    <kendo-combobox
      v-model="keyword"
      :placeholder="'ค้นหาด้วยชื่อลูกค้า'"
    ></kendo-combobox>

    <GridProxy
      ref="grid"
      :columns="columns"
      :url="'/Customer/Search'"
      :parameters="{ keyword }"
    />
  </div>
</template>

<script>
import { sync } from "vuex-pathify";
import MainActionToolbar from "faco.core@shared/Controls/Toolbar/MainActionToolbar";
import GridProxy from "faco.core@shared/Controls/Grid/GridProxy";

export default {
  components: { MainActionToolbar, GridProxy },
  props: {
    mode: { type: String, default() { return ""; } }
  },
  computed: {
    ...sync("Demo_SearchCustomer/*")   // ต้องตรงกับชื่อ module ที่จะได้จากขั้นตอนที่ 2
  },
  data() {
    return {
      keyword: "",
      columns: [
        { field: "code", title: "รหัสลูกค้า" },
        { field: "name", title: "ชื่อลูกค้า" }
      ]
    };
  },
  methods: {
    onSearch() { this.$refs.grid.onPageLoad(1); },
    onReset() { this.keyword = ""; }
  },
  mounted() {
    this.$store.set("Layout_MasterLayout/setTitle", {
      titleName: "ค้นหาลูกค้า",
      subTitleName: "Demo > ค้นหาลูกค้า"
    });
  }
};
</script>
ทันทีที่บันทึกไฟล์นี้...
route /Demo/SearchCustomer/Index จะถูกสร้างขึ้นเองอัตโนมัติ (บทที่ 07) — ไม่ต้องไปแก้ router/index.js เลย แค่รอ webpack-dev-server compile เสร็จแล้วเข้า URL นั้นได้ทันที

ขั้นตอนที่ 2 — สร้าง Vuex Store module คู่กัน

สร้างไฟล์ src/app/store/views/Demo/Stores/SearchCustomer.js (ต้องอยู่ใต้โฟลเดอร์ Stores/ เสมอ ไม่งั้นจะไม่ถูก auto-register)

import { make } from 'vuex-pathify';
import axios from "faco.core@repository";

function state() {
  return {
    dataInGrid: [],
    totalItems: 0
  };
}

const getters = {
  ...make.getters(state)
};

const mutations = {
  ...make.mutations(state)
};

const actions = {
  ...make.actions(state),
  searchCustomer({ commit }, payload) {
    return new Promise(resolve => {
      axios.financeApi.get('/Customer/Search', { params: payload })
        .then(response => {
          commit('dataInGrid', response.data.content);
          commit('totalItems', response.data.total);
          resolve(true);
        })
        .catch(() => resolve(false));
    });
  }
};

export default { namespaced: true, state, getters, actions, mutations };
เช็คชื่อ module ให้ตรงกันเป๊ะ ๆ
ไฟล์นี้อยู่ที่ store/views/Demo/Stores/SearchCustomer.js → ชื่อ module ที่ auto-register จะเป็น Demo_SearchCustomer ต้องตรงกับที่เขียนใน sync("Demo_SearchCustomer/*") ในไฟล์ view ขั้นตอนที่ 1 เป๊ะ ๆ (ตัวพิมพ์ใหญ่-เล็กก็ต้องตรง) ไม่งั้น sync จะไม่เจอ module และเงียบ ๆ ไม่มี error ชัดเจน

ขั้นตอนที่ 3 — เพิ่มคำแปลภาษา (ถ้ามีข้อความใหม่)

ข้อความ label/ปุ่มควรใช้ this.$t("KEY") แทนการ hardcode ข้อความไทย/อังกฤษตรง ๆ เพื่อรองรับสองภาษา — ไปเพิ่ม key ใหม่ในไฟล์ locale ที่ src/locales/

ขั้นตอนที่ 4 — รันดูผลจริง

# เลือก script ให้ตรงกับ module ที่กำลังทำงานอยู่ (ดูบทที่ 06)
npm run local            # ทุก module
npm run local-finance    # เฉพาะ Finance
npm run local-main       # Finance,HPAdmin,Legal,Collection

เข้า URL ตามโครงสร้างไฟล์ที่สร้าง: http://localhost:8080/Demo/SearchCustomer/Index

Checklist สรุปทั้งหมด

  • สร้าง .vue ใต้ src/app/views/<Domain>/<ScreenName>/Index.vue — route จะเกิดเองตาม path นี้
  • สร้าง .js ใต้ src/app/store/views/<Domain>/Stores/<ScreenName>.js — ตั้งชื่อให้ตรงกับ path เพื่อให้ sync() ใน view เจอ module
  • ใช้ MainActionToolbar แทนปุ่ม save/reset เอง, ใช้ GridProxy แทน grid+datasource เอง
  • เรียก API ผ่าน faco.core@repository เท่านั้น (ได้ loading + auth header ให้ฟรี)
  • ตั้งชื่อ title หน้าจอผ่าน $store.set("Layout_MasterLayout/setTitle", {...}) ใน mounted()
  • ใช้ $t("KEY") สำหรับข้อความที่ต้องรองรับ 2 ภาษา แทนการ hardcode
  • รับค่าจาก URL ผ่าน props mode/modeFlag/modePage แทนการอ่าน $route.query เอง (ถ้าเป็น query มาตรฐานของระบบ)
  • ถ้าต้องการ dialog/alert ใช้ AlertMessage/kendo-window ที่มีอยู่แล้ว ไม่ประกอบ Kendo Window เอง
ติดตรงไหน ให้ทำแบบนี้
1) หา pattern คล้าย ๆ กันใน src/app/views/Demo/ หรือ module ที่ใกล้เคียงที่สุดก่อน (เช่นถ้าทำหน้า Finance ให้ดูตัวอย่างใน src/app/views/Finance/)
2) เปิดไฟล์ shared component ตรง ๆ ใน src/app/shared/ เพื่อดู props/events ที่รองรับจริง (props ที่เขียนในบทเรียนนี้เป็นตัวอย่างหลัก ๆ อาจไม่ครบ 100%)
3) กลับมาอ่านบทที่ 07 (Router & Vuex ของโปรเจกต์) ซ้ำถ้าเจอปัญหาเรื่อง store module ไม่เจอ/undefined
